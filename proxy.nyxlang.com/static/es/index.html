Config simple

Un archivo para todo

Define upstreams, certificados TLS y limites en un solo TOML.

# proxy.toml
[server]
listen      = 443
workers     = 64
tls_cert    = "/etc/letsencrypt/live/ejemplo.com/fullchain.pem"
tls_key     = "/etc/letsencrypt/live/ejemplo.com/privkey.pem"
rate_limit  = 100
access_log  = "/var/log/nyx-proxy/access.log"

[upstream.0]
name        = "web"
host        = "127.0.0.1"
port        = 3000
hostname    = "ejemplo.com"
Caracteristicas

Todo lo que necesitas

TLS multi-dominio con SNI
Sirve multiples dominios en una sola IP. Cada dominio con su propio certificado TLS. Actualmente sirviendo 4 dominios.
Health checks
Verificaciones TCP activas cada N segundos. Backends caidos se remueven automaticamente y se restauran al recuperarse.
Access logs
Formato estructurado: timestamp, method, path, status, latencia. Grep-friendly, escrituras protegidas con mutex.
Rate limiting
Contador de ventana deslizante por IP. Retorna HTTP 429 con header Retry-After. Configurable en proxy.toml.
Connection pooling
HTTP keep-alive entre proxy y backends. Reduce overhead de conexiones.
Redirect HTTP
Redireccion automatica HTTP a HTTPS en puerto 80.

Comparacion

Reemplaza Nginx

Para setups simples de proxy inverso, nyx-proxy hace lo mismo en una fraccion del tamano.

Nginx
~1.5MB binario. Configuracion compleja. Sistema de modulos. Docenas de dependencias.
nyx-proxy
~170KB binario. Config TOML simple. TLS, health checks, rate limiting, access logs integrados. Cero dependencias.
