Qué hace

Los fundamentos, bien hechos

Todo lo que un editor de terminal necesita para sentirse sólido — los detalles que separan un juguete de algo en lo que realmente escribirías.

SIGWINCH

Resize en vivo

Arrastrá el borde de la terminal y el buffer se re-renderiza al instante — sin necesidad de apretar una tecla.

?2026

Render sin parpadeo

Una escritura bufferizada por frame dentro de una región de output sincronizado. Sin tearing, sin flash — incluso sobre un SSH lento.

UTF-8

Consciente de codepoints

El movimiento del cursor, la edición y el render trabajan en codepoints, no bytes — los caracteres de 2 a 4 bytes simplemente funcionan.

SGR mouse

Click, arrastre, scroll

Click para posicionar el cursor, arrastre para seleccionar, rueda para scrollear. Se apaga limpio en cada salida.

Ctrl+Z / Ctrl+Y

Undo/redo incremental

Deltas por operación con coalescing de tipeo, hasta 1000 niveles — una ráfaga de tipeo colapsa en un solo undo.

bracketed paste

El paste queda literal

Los pastes multilínea se insertan tal cual como un solo grupo de undo — sin auto-indent arruinando tu código.

?1049

Pantalla alternativa

Tu scrollback queda intacto al salir, como vim o less. El editor vive en su propia pantalla.

Ctrl+F / Ctrl+G

Búsqueda e ir-a-línea

Búsqueda incremental con wrap-around, y salto directo a un número de línea. La navegación que buscás.

defer cleanup

Siempre restaura la terminal

Termios, la pantalla alternativa y los modos de mouse se deshacen vía defer en cada salida — sin shell arruinado tras un crash.

Ponelo a andar

Instalá y abrí un archivo

nyx-edit es un programa Nyx — instalá el toolchain, compilá el único binario, corrélo. Sin paquete que gestionar.

$ curl -sSf https://nyxlang.com/install.sh | sh
$ git clone https://github.com/nyxlang-dev/nyx-edit
$ cd nyx-edit && nyx build
$ ./nyx-edit README.md
# abre README.md — flechas para moverte, Ctrl+S para guardar, Ctrl+Q para salir

Por qué existe

Una prueba de estrés para el lenguaje

Un editor de terminal no perdona: input de bytes crudos, secuencias de escape, manejo de señales, loops de render ajustados. nyx-edit es un archivo de Nyx que hace todo eso — el mismo lenguaje que escribe el compilador, el KV store y el framework web.

Nyx

El lenguaje auto-hospedado en el que está escrito nyx-edit. Compila a binarios nativos chicos sin runtime.

 nyx-serve

El framework web — WebSockets, plantillas, sesiones. El lado servidor del mismo toolchain.

 nyx-kv

Almacén clave-valor compatible con Redis. El lado de los datos del ecosistema.
