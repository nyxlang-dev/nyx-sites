De cero a una app full-stack desplegada

Este tutorial construye una sola cosa, paso a paso: una aplicación de notas. Arranca como una API JSON que pruebas con curl, gana middleware, sesiones, páginas renderizadas en el servidor y WebSockets, y termina desplegada detrás de systemd con una base de datos real — nyx-kv. Cada bloque de código de estas páginas se compiló y corrió contra el release actual antes de publicarse.

Qué necesitas: una máquina Linux o macOS con un toolchain de C (clang), y 15–40 minutos según hasta dónde llegues.

El recorrido

  1. Instalar — un comando, verifica con nyx --version.
  2. Crear un proyecto — nyx init, declare the dependency, understand the layout.
  3. Primera API — rutas, parámetros, input de forms, JSON de salida; prueba con curl.
  4. Middleware y sesiones — the status-0 contract, CORS, login with cookie sessions.
  5. Plantillas, estáticos y forms — escape-by-default rendering, cached static files.
  6. WebSockets — rutas app_ws con parámetros {room}.
  7. Full-stack — sesiones + plantillas + almacenamiento en nyx-kv, desplegado con systemd.
¿Ya conoces lo básico? La referencia de la API vive en el repo: docs/API.md — cada función pública con su firma exacta.
